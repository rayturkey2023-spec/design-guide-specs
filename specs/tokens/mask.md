# 蒙层 Token — Mask

> 模块：`tokens/mask`  
> 分类：基础组件 / FOUNDATIONS  
> 适用平台：移动端 / PC 端

---

## 概述

蒙层（Mask）用于弹窗、浮层、半浮层等场景的背景遮罩，通过透明度控制视觉焦点强度。规范分为黑色系蒙层与白色系蒙层两类。

---

## 颜色 Token

### 黑色系蒙层

| Token 名称 | 颜色值 | 透明度 | RGBA Hex | 使用场景 |
|---|---|---|---|---|
| `mask-black-30` | `#000000` | 30% | `#0000004D` | 轻度遮罩，底层内容高度可见 |
| `mask-black-60` | `#000000` | 60% | `#00000099` | 中等遮罩，搜索/筛选/编辑场景 |
| `mask-black-80` | `#000000` | 80% | `#000000CC` | 较强遮罩，下单页/支付页弹层 |
| `mask-black-90` | `#000000` | 90% | `#000000E6` | 强遮罩，高压提示/强打断弹层 |

### 白色系蒙层

| Token 名称 | 颜色值 | 透明度 | RGBA Hex | 使用场景 |
|---|---|---|---|---|
| `mask-white-30` | `#FFFFFF` | 30% | `#FFFFFF4D` | 亮色页面轻度蒙层 |
| `mask-white-60` | `#FFFFFF` | 60% | `#FFFFFF99` | 亮色页面中等遮罩 |
| `mask-white-80` | `#FFFFFF` | 80% | `#FFFFFFCC` | 亮色页面强遮罩 |

---

## 使用规则

### 透明度约束

```
最低透明度：30%（不允许低于此值，否则遮罩效果失效）
最高透明度：90%（不允许高于此值，避免影响系统一致性）
允许范围：30% ～ 90%
```

### 场景化透明度选择

| 业务场景 | 推荐 Token | 说明 |
|---|---|---|
| 订单弹窗 / 表单输入 | `mask-black-90` | 强打断，突出当前输入区域 |
| 首页站内提示 | `mask-black-90` | 强引导，吸引用户注意 |
| 列表/搜索筛选面板 | `mask-black-60` | 中等遮罩，保留页面感知 |
| 购物车商品选择 | `mask-black-60` | 中等遮罩，保留底层商品感知 |
| 下单页/支付浮层 | `mask-black-80` | 较强遮罩，突出金额与操作按钮 |
| 购物车商品编辑 | `mask-black-80` | 较强遮罩，聚焦编辑区域 |
| 优惠券/促销弹层 | `mask-black-90` | 最强遮罩，完全聚焦促销内容 |

## 交互规范

### 点击行为

蒙层点击行为需与弹层类型配合定义：

- **可点击关闭**：信息提示类弹层、Toast 背景蒙层，点击蒙层区域可关闭
- **不可点击关闭**：强操作确认类弹层（如支付、删除），点击蒙层不响应，需用户明确操作关闭按钮

> 具体关闭策略以各组件规范为准，蒙层本身不独立定义关闭行为。

### ADA 点击可访问性

符合 WCAG 2.1 / ADA 标准的点击行为要求：

- **蒙层不可点击：** 蒙层点击时不可触发关闭蒙层上方控件的交互，如关闭弹窗，必须点击关闭按钮才能关闭控件
- **键盘可访问**：蒙层覆盖时，`Tab` 焦点应被限制在弹层内部（Focus Trap），不允许焦点穿透到蒙层下方内容
- **Escape 键关闭**：对于可关闭的蒙层，必须支持按 `Esc` 键关闭，不能仅依赖点击蒙层区域
- **屏幕阅读器**：蒙层激活时，底层页面内容应对屏幕阅读器隐藏（`aria-hidden="true"`），防止读出被遮罩的内容
- **角色标注**：蒙层容器应设置 `role="dialog"` 或 `role="alertdialog"`（强打断场景），并配合 `aria-modal="true"`
- **焦点回归**：弹层关闭后，焦点必须回到触发该弹层的元素上，不允许焦点丢失

---

## 可访问性（Accessibility）

- 蒙层不能过浅（<30%）：无法形成焦点，遮罩意义丧失
- 蒙层不能过深（>90%）：可能影响系统整体可读性与一致性
- 蒙层颜色应与弹层内容形成足够对比度，确保弹层文字/按钮的可读性
