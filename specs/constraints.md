# 全局禁止事项

> 本文件每次调用设计规范时必须加载，所有条目为硬性约束。

## Token 使用

- 禁止使用规范 Token 以外的颜色值（如 #333333、#f0f0f0 等非 Token 色）
- 禁止使用规范 Token 以外的间距值（如 10px、15px、22px）
- 禁止直接使用像素硬编码替代 Token（Figma 中应引用 Token 变量）

## 组件使用

- 禁止同屏放置 2 个 Primary 按钮
- 禁止弹窗内嵌套弹窗
- 禁止手动绘制与组件库已有组件外观相似的自定义元素（必须使用组件实例）
- 禁止在移动端使用 PC 端专属组件（Tooltip、复杂多列 Table 等）
- 禁止忽略组件的 Disabled / Loading / Empty 状态设计

## 状态设计

- 所有列表页面必须包含：加载中态（Skeleton）、空态、错误态
- 所有表单提交按钮：未满足必填条件时必须为 Disabled 态
- 禁止用全屏 Loading Spinner 替代骨架屏（列表类页面）

## 交互反馈

- 操作成功反馈用 Toast，需用户确认的用 Modal，表单错误用行内提示
- 破坏性操作（删除、清空、注销）必须有二次确认弹窗
- 禁止弹窗或浮层自动弹出（必须由用户操作触发）

## 跨端

- 禁止在移动端直接套用 PC 端布局，反之亦然
- 多端适配方案需单独设计，不做拉伸缩放处理
