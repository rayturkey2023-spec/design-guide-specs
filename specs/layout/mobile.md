# 移动端布局规范

## 基础页面结构

```
屏幕（全高）
├── 状态栏（系统层，不参与设计）
├── 导航栏
│   ├── 高度：【填入 px，建议 44px】
│   ├── 背景：color-bg-card
│   ├── 左区：返回按钮 / 关闭按钮
│   ├── 中区：页面标题（text-heading-md）
│   └── 右区：文字操作 或 图标操作（最多 2 个）
├── 内容区
│   ├── 背景：color-bg-page
│   ├── 左右边距：spacing-16
│   └── 顶部与导航栏间距：spacing-0（紧贴）或 spacing-8
└── 底部区（按需选一种）
    ├── 底部导航（Tab Bar）：高度 【填入 px，建议 50px】+ 系统安全区
    ├── 固定操作按钮：高度 【填入 px，建议 80px，含安全区】，背景 color-bg-card
    └── 无底部操作：内容区底部留 spacing-24 + 系统安全区
```

## 内容区层级规则

- 页面背景（color-bg-page）→ 卡片（color-bg-card）→ 浮层（含遮罩）
- 卡片不在卡片内直接嵌套，同层级并列展示
- 分组内容用卡片区块包裹，卡片间距 spacing-12

## 触控规范

- 可点击元素最小触控区域：44×44px
- 列表项最小高度：【填入 px，建议 48px】
- 图标按钮的点击区域需内边距补足至 44×44px（即使图标本身更小）

## 安全区处理

- 底部固定操作区必须增加 iOS 安全区高度（env(safe-area-inset-bottom)）
- 全面屏设备底部内容不被遮挡
